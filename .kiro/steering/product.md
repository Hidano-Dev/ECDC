# プロダクト概要

ECDC (Equipment Configuration Diagram Creator) は、バーチャルプロダクション案件における機材構成図の作成を支援する個人ツール。機材台帳（Google Sheets）と配線GUI を組み合わせ、draw.io 形式の構成図を自動生成するワークフローを提供する。

## コア機能

1. **機材台帳の一元管理** -- Google Sheets を Single Source of Truth として機材マスタを管理する
2. **配線GUI による接続定義** -- Web アプリ上で機材をキャンバスに配置し、ドラッグ操作で接続線を定義する
3. **会場空間の構造化** -- 会場図面から YAML で空間情報（ゾーン、隣接関係、ケーブル敷設距離）を表現する
4. **draw.io XML の自動生成** -- 案件定義 YAML + 会場定義 YAML から Claude Code が構成図を生成する

## 対象ユースケース

- バーチャルプロダクション案件の機材構成図を作成する場面
- 50台以上の機材と80本以上の接続を持つ大規模構成の効率化
- 過去案件の機材構成を再利用して新規案件の構成図を素早く立ち上げる
- 会場の地理的条件（フロア構成、ケーブル敷設距離）を加味したレイアウト支援

## 提供価値

- **二重メンテの排除**: 機材リストと構成図が別管理になる問題を解消し、台帳からの自動反映を実現する
- **作業時間の短縮**: draw.io 上でゼロから配置・接続する手作業を GUI + 自動生成に置き換える
- **空間情報の構造化**: 会場のフロア構成やケーブル敷設経路を YAML で管理可能にし、LLM によるレイアウト支援を可能にする
- **再利用性**: 会場定義 YAML と案件定義 YAML を分離することで、同一会場の別案件への再利用を容易にする

## ワークフロー概要

```
会場図面 -> Claude Code -> 会場定義YAML
機材登録(Google Sheets) -> 配線GUI -> 案件定義YAML
案件定義YAML + 会場定義YAML -> Claude Code -> draw.io XML -> 手動微調整
```

## 利用者・運用形態

- 個人利用（1名）のため、認証・権限管理は不要
- 不定期利用（案件ごと）
- ローカル実行が基本（将来的に Vercel 等へのデプロイも可能）
